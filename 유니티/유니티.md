**1.유니티 네트워크 라이브러리**

**(1)Unity Netcode for GameObjects(NGO)**
- 유니티의 표준 공식 기술, 현대적인 고수준(High-level) 아키텍처를 가지고 있음.

(1)내부 구조
   (a) 아키텍처
	- 고수준API - 사용자가 소켓이나 바이트 단위 데이터를 직접 다루지 않도록 추상화되어있음
	- Messaging System - 유니티의 Unity Trasport 레이어 위에서 작동하며, 내부적으로 FastBufferWriter/Reader를 사용하여 데이터를 직렬화해 패킷으로 보내며 리플렉션을 쓰지 않아 런타임 성능이 준수함.
   (b) 핵심 데이터 모델
- NetworkManager - 상태(State) 중심의 모든 네트워크 설정을 관리하는 싱글톤 허브. 변수의 현재 값을 동기화해 새로운 유저가 들어왔을 때 '현재 상태'를 즉시 알 수 있어 편리함.
- RFC(Remote Procedure Call) - 이벤트(Event) 중심이며 특정 함수를 상대방 기기에서 실행시킴
  


**(2)Mirror(미러)**
- 가장 오래된 라이브러리로 구조가 단순하고 직관적이며 오픈 소스라 내부 로직을 커스텀하기에 좋
