**스택(Stack)** - 같은 구조와 같은 크기의 자료를 정해진 방향으로만 쌓음. 
**Top**으로 정한 곳으로만 접근가능
가장 마지막에 삽입된 자료가 가장 먼저 삭제됨

+후입선출(LIFO, Last In First Out)
 삽입연산-'push'
 삭제연산-'pop',
 
 **큐(Queue)** - 한쪽에서는 삽입작업,다른 한쪽에서는 삭제 작업이 이루어짐
 삭제연산이 되는곳을 프론트(frot)
 삽입연산이 되는곳을 리어(rear)라고 칭함.
 
 +*선입선출*(FIFO, First in first out)
  삽입연산-인큐(enQueue)
  삭제연산-디큐(dnQueue)
  
  **맵(map)** - 키(Key)와 값(Value)을 한 쌍으로 묶어서 저장하는 자료구조. (파이썬의 딕셔너리처럼)
  
  구조 - **균형 이진 탐색 트리 $O(log N)$  
  데이터가 들어올 때 마다 키(Key)를 기준으로 자동으로 오름차순 정렬을 함.**
  메모리 사용량 - **상대적으로 적음**
  
  언제 쓰는지?
  - 데이터를 정렬된 상태로 유지해야 할 때 (ex) 사전 순으로 출력해야 할 때
  - 특정 범위의 데이터를 찾아야 할 때
  
  **unordered_map** - 정렬이 되지 않은 map
  
  구조 - **해시 테이블**
  속도 - **바로 찾아가므로 평균 O(1). 단, 해시 충돌이 발생하면 $O(N)$이 될 수 있음.**
  메모리 사용량 - **상대적으로 많음(해시 테이블 공간 필요)**
  
  언제 쓰는지?
  - 정렬은 알 바 아니고, 무조건 데이터를 가장 빠르게 넣거나 찾아야 할 때
  
  ex) std::unordered_map<std::string, std::string> passwords;
