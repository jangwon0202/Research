**(3)채널 패킹**  **-중요함!!-**
픽셀 하나는 4개의 방으로 나뉨
- R(Red) - 8비트(0~255 or 0.0 ~ 1.0) - 빨강
- G(Green) - 8비트(0~255 or 0.0 ~ 1.0) - 초록
- B(Blue) - 8비트(0~255 or 0.0 ~ 1.0) - 파랑
- A(Alpha)) - 8비트(0~255 or 0.0 ~ 1.0) - 투명도

+Alpha는 투명도(0이면 투명, 1이면 불투명)

Color.png 라는 파일을 만들면 채널에 R,G,B,A라는 4개의 빈 방(흑백 레이어)가 생김
FHD(1920X1080) 이미지면 픽셀은 1920 X 1080 = 2,073,600개만큼 생김
픽셀 1개당 데이터 크기는 4바이트(R 1바이트, G 1바이트, B 1바이트, A 1바이트)
즉 최종 데이터 크기는 2,073,600 X 4바이트 = 8,294,400 바이트 (약 8.3MB))

각 픽셀마다 R,G,B,A 라는 4개의 **구조체 변수**가 들어있음
0.0(검은색) - 해당 칸의 전구를 완전히 끈 상태
0.5(회색) - 해당 칸의 전구를 중간 밝기로 킨 상태
1.0(하얀색) - 해당 칸의 전구를 최대 밝기로 킨 상태

R 방만 떼서 봄(거친 노이즈를 만든다고 가정)
$[0,0]$의 R값을 0.1, $[0,1]$의 R값을 0.8, $[0,2]$의 R값을 0.2으로 확 튀게 하는 식으로 거칠게 표현함

G 방만 떼서 봄(부드럽게 그라데이션을 만든다고 가정)
$[0,0]$의 R값을 0.9, $[0,1]$의 R값을 0.8, $[0,2]$의 R값을 0.75으로 은은하게 어두워지는 식으로 표현.

불꽃 이펙트를 만든다고 가정했을 때
이런식으로 머티리얼 내에서 R핀을 뽑으면 거칠게 불꽃이 튀는걸 표현, G핀을 뽑으면 부드럽게 퍼지는 불꽃의 중심 부분을 표현할 수 있음

```
// GPU 비디오 메모리(VRAM) 및 셰이더 내부에서 실제 구동되는 픽셀 구조
struct FColorPixel
{
	float R; // 🔴 R 핀 파이프라인 (0.0 ~ 1.0 수치 데이터)
	float G; // 🟢 G 핀 파이프라인 (0.0 ~ 1.0 수치 데이터)
	float B; // 🔵 B 핀 파이프라인 (0.0 ~ 1.0 수치 데이터)
	float A; // ⚫ A 핀 파이프라인 (0.0 ~ 1.0 수치 데이터)
};
FColorPixel VRAM_Texture_Array[1920 * 1080]; // FHD 이미지 1장의 본질 (약 8.3MB)
```

![[Pasted image 20260531030730.png]]

R핀 소켓에서 선을 뽑으면 ? -> Pixel.R 값만 추출(거친 데이터)
G핀 소켓에서 선을 뽑으면 ? -> Pixel.G 값만 추출(그라데이션 데이터)

- 연산 예시(불타는 구체)
	- R핀 선 - > Multiply(핑크색) -> 최종 노드의 이미시브 컬러(스스로 빛나는 옵션)에 링크
	- G핀 선 -> 최종 노드의 러프니스 (표면 거칠기 옵션)에 링크
- 빛의 합성(노란색)
	그래픽스(빛의 합성)에서 노란색은 빨강과 초록의 합성이므로 R:1.0 + G:1.0이다. 두 채널의 하얀색(1,0) 데이터가 겹치는 좌표는 자동으로 눈부신 노란색 화염 코어가 됨.

**(4)마스터 머티리얼**
기법을 총동원해서 어떤 이펙트나 물체든 다 만들어낼 수 있게 뼈대를 만들어둔 템플릿

보통은 기능별 모듈로 쪼개서 설계함

<4단계 모듈 구조>

- 1단계 - 텍스쳐 좌표 변형 모듈(Panner & Rotator) - UV 변형 구역
	-> 가만히 멈춰있는 흑백 노이즈 지도를 살아 움직이게(흐르게) 만드는 연산 구역
	 
	핵심 노드 : Time(시간) , Panner(패너), Rotator (회전)
	 
	**매커니즘**
	 - 이미지 파일의 좌표계(UV)에 실시간으로 흘러가는 **시간 값**을 더하거나 곱해주는 단계
	 - 마스터 머티리얼에서는 각 채널(R, G, B, A)마다 흐르는 속도와 방향을 다르게 제어할 수 있도록 벡터 변수(파라미터)를 맨 앞에 세팅해둠.

- 2단계 - 패킹 데이터 추출 모듈(텍스처 샘플링 구역)
	->1단계에서 마구 움직인 좌표를 기반으로, Color.png 파일에서 R,G,B,A 각각의 흑백 데이터(0.0~1.0)를 분리해 추출하는 구역
	
	핵심 노드 : Texture Sample(텍스처 샘플)
	
	**매커니즘 예시** 
	- 하나의 이미지 노드에서 마우스 드래그로 선을 따로따로 뽑아내는 구조
	- 🔴 **R 핀:** 자잘하고 거친 노이즈 수치 데이터 추출 $\rightarrow$ (ex: 불꽃 잔재용)
	- 🟢 **G 핀:** 뭉툭한 원형 그라데이션 수치 데이터 추출 $\rightarrow$ (ex: 불꽃 코어용)
	- 🔵 **B 핀:** 또 다른 형태의 무늬 추출 $\rightarrow$ (ex: 연기 또는 공기 굴절 왜곡용)
	- ⚫ **A 핀:** 외곽을 깔끔하게 잘라낼 형태의 마스크 수치 추출 -> (ex:  투명도 제어용)

- 3단계 - 수학적 가중치 및 마스킹 모듈(데이터 정제 구역)
	->2단계에서 뽑아낸 날 것의 데이터 숫자들을 수학 공식을 이용해 다듬는 구엽
	
	핵심 노드 : Multiply(곱하기), Add(더하기), OneMinus (반전, 1-x), Clamp(수치 제한)
	
	**매커니즘 예시**
	- R 핀에서 나온 자잘한 노이즈가 너무 흐릿하면, Multiply 노드로 명암 대비를 조절
	- R 핀(거친 노이즈)과 G핀(원형 마스크)을 서로 곱해서 "구체 중심부에서만 노이즈가 선명하게 살아남고, 외곽으로 갈수록 자연스럽게 소멸(0.0이 됨)하게" 만드는 형태의 데이터 마스킹 연산을 처리함

- 4단계 - 최종 색상 합성 및 출력 모듈(기능 부여 구역)
	->3단계에서 마무리된 최종 무늬 수치 데이터에 진짜 색상(Vector3)을 입히고, 엔진의 하드웨어 옵션 소켓에 골라 꽂는 마무리 단계
	
	핵심 노드 : Lerp(선형 보간), 최종 마스터 머티리얼 출력 노드
	
	**매커니즘 예시**
	
	(1) 🎨 물체의 기본 외형 구역
		베이스 컬러(Base Color) : 물체 표면의 순수한 고유 색상 데이터(빛이나 그림자 X)
			입력 데이터 : Vector3(RGB 값)
			+ 핑크색 구체를 만들고 싶다면 (1, 0, 0.5)
		이미시브 컬러(Emissive Color) : 물체가 스스로 빛을 내뿜는 영역과 색상
			입력 데이터 : Vector3(RGB 값)
			+ Multiply로 10이나 100같은 숫자를 곱하면 훨씬 빛나게 할 수 있음.
	
	(2) 💎 재질감 설정 구역(PBR 렌더링의 핵심) - **물체의 물리적 재질을 결정!!**
		메탈릭(Metallic) : 표면이 금속이냐 비금속이냐를 결정하는 스위치
			+ 0.0 = 돌, 플라스틱 등(비금속) / 1.0 = 철, 구리, 금 등(금속)
		러프니스(Roughness) : 표면의 거칠기. 빛이 얼마나 사방으로 번지는지 결정함.
			+ 0.0 = 거울이나 유리처럼 매끄러움(번쩍임) / 1.0 = 석고나 천처럼 까칠함(빛이 완전히 번짐)
		스페큘러(Specular) : 비금속(Metalli)일 때, 빛이 반사되는 강도.
			+ 기준값은 0.5라 웬만하면 안 건드리지만, 플라스틱의 반사율을 미세하게 조절할 때 사용.
		애니소트로피(Anisotropy)
			- 빛이 정원형이 아니라 가로나 세로로 길게 늘어져서 반사되는 성질(**이방성 반사**)
			+ LP판의 뒷면이나 머리카락 결, 결이 살아있는 알루미늄 판같은 특수 재질을 표현
	
	(3) 👻 투명도 및  시스루 구역(이펙트 필수)
		(1)오파시티(Opacity) : 물체를 투명하게 깎아냄.
			+ 0.0 = 완전 투명(안 보임) / 1.0 = 완전 불투명
			+ Opacity Mask는 투명/불투명 두 가지만 판정할 때
		서브서피스 컬러(Subsurface Color)
			- 빛이 물체 표면을 뚫고 들어가 내부에서 투과되어 나오는 색상(피부, 양초, 나뭇잎)
			+ 역광을 받았을 때 사람 귀가 빨갛게 비치는 현상 등 내부 혈색을 지정해 주는 핀
		
	(4) 📐 표면 굴곡 및 디테일 구역
		(1)노멀(Normal)
			- 폴리곤(정점)을 실제로 늘리지 않고, 눈속임으로 입체적인 주름이나 굴곡을 만드는 핀
		+바위의 거친 표면이나 벽돌 틈새 같은 디테일을 연산할 때 필수
		(2)탄젠트(Tangent) 
			- 노멀 맵이 기준점으로 삼는 표면의 가로축 방향(접선) 데이터
			+애니소트로피(이방성) 연산할 때 머리카락 결의 방향을 알려주기 위해 노멀과 세트로 꽂아주는 경우가 많음
	(5) 🌪️ 수학적 왜곡 및 위치 변형 구역 (고급 기법)
		(1)월드 포지션 오프셋(World Position Offset, WPO)
			- GPU 연산으로 물체의 정점(Vertex)좌표를 강제로 움직여서 형태를 일그러뜨리는 핀
			+ C++ 코드가 액터 전체를 움직인다면, WPO는 셰이더 안에서 나무가 바람에 흔들리거나, 깃발이 펄럭이거나, 마법 구체가 출렁거리며 찌그러지는 연출 등 
		(2)리프랙션(Refraction)
			- 빛에 물체를 통과할 때 꺾이는 굴절률
			+ 물속을 볼 때 굴절되거나, 돋보기 렌더링, 아지랑이 효과(Distortion) 등 사용
	(6) 👁️ 특수 연산 및 화면 셰이딩 구역
		(1)앰비언트 오클루전(Ambient Occlusion, AO)
			- 구석진 틈새나 골짜기에 자연스러운 섀도우(그림자)를 미리 구워 넣는 소켓
			+ 0.0 = 틈새라서 완전 어두움 / 1.0 = 평평해서 빛을 잘 받음
		(2)픽셀 뎁스 오프셋(Pixel Depth Offset, PDO)
			- 카메라와의 거리(깊이) 판정을 인위적으로 뒤로 밀어버리는 기능
			+돌멩이가 땅에 박힐 때 경계선이 칼로 자른 듯 딱딱하게 겹치지 않고, 모래로 부드럽게 스며들듯 교차하게 만들 때 쓰는 프로급 최적화 핀
		(3) 정면 머티리얼 (Front Material) / 오파시티 오버라이드(Opacity Override)
			- 5.4 버전 이후 도입된 Strata / Substrate 시스템에서 사용하는 핀
			+ 기존의 고정된 소켓 방식 대신 여러 재질(ex: 철 위에 투명 코팅을 한겹 더 얹는 구조)을 레이어 구조체 형태로 묶어서 한 번에 출력할 때 쓰는 고난도 핀.
			  

요약 정리!

- 투명하게 할 때는 **오파시티**
- 출렁이게 할 때는 **월드 포지션 오프셋**
